Getting Started

This guide will walk you through buiding a series of “sketches” (i.e. little web pages) that respond to different user inputs, namely mouse movements, keystrokes, and the user’s camera. By the end you will have a grasp of basic programming concepts and how they support interactivity on the web.

What is a web page?

For the purposes of this guide, you can think of a web page as an interactive thing made of three parts: HTML, CSS, and JavaScript. Roughly speaking, HTML defines the “content” of the page, CSS defines how the page “looks,” and JavaScript controls how the page “behaves.” Your little web pages will be all about behavior, so you will be dealing almost exclusively with that third part, JavaScript.

JavasScript is a powerful programming language that forms the backbone of every interaction you have with any website, and, as you can imagine, it takes a long time to master. Luckilly, developers have created many, many free JavaScript “libraries” you can use to solve specific kinds of problems. Whatever you want to do, chances are somebody has already done something similar, and (in general) web developers love sharing their solutions.

What is p5.js?

p5.js is a creative coding library built in javaScript. It’s not the only way to do creative coding, but it has a few notable benefits. First, it’s open source and works in any web browser, meaning it is free and easy to use on the internet. Second, p5.js is popular, so finding examples and tutorials for most problems is relatively easy. Third, it has a proven track record introducing artists and designers to software development.

What is a “sketch”?

Projects in p5.js are called “sketches,” and each one begins by defining a setup() and a draw() function. For now, you can think of a function as a reusable task. Some functions are defined by JavasScript itself, some by the web browser, some by p5, and others are defined by you. The first thing p5 does when your page loads is find the setup function (defined by you) and call it (i.e. make it do the task). Then, p5 will find your draw function and begin calling it repeatedly an animation loop, simulating continuous motion like frames in a video.

Inisde the body of your setup and draw functions—body refers to the code between the curly braces {...}—you will make use of variables, constants, and functions defined by p5 in order to create your sketch. A variable is a value that could change over time, like the x/y position of the mouse or the width of the canvas. A constant is a value that never changes, and you will notice p5 constants are always ALL CAPS. Functions, as mentioned above, define a sharable task, like drawing a circle or rounding a number. Building on our basic structure, here is a simple sketch that uses three functions and two variables defined by p5.

This setup creates a canvas 600px wide by 400px tall. It does so by calling p5’s createCanvas function. A Function is called by writing its name followed by parentheses (). Inside the parentheses you pass any arguments the function expects, separated by commas. In this case, createCanvas expects two number arguments, the width of the canvas followed by its height.

Inside the draw function, the first call is to p5’s background function with the string argument 'magenta', which fills the whole canvas with white. You can represent colors several ways in p5, but don’t worry too much about it for now. After we fill the background, we draw a circle. The circle function accepts three number arguments: the “x” position of the circle 300, the “y” position of the circle 200, and the diameter of the circle 200. The x/y positions and diameter refer to the cooridinate system p5 uses to draw, which you will learn about later.
